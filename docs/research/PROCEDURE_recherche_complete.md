# Procedure - Recherche complete

Cette procedure sert a produire une recherche SAFE longue, claire et peu sujette aux hallucinations.

## Quand l'utiliser

Utiliser cette procedure pour:

- nouveaux domaines de pratique
- sujets reglementaires sensibles
- etudes de marche ou de positionnement
- dossiers clients complexes
- recherches fondatrices qui doivent servir de reference reutilisable

## Etapes

### 1. Cadrer la question

Definir:

- la question exacte
- le perimetre
- le public destinataire
- la decision que la recherche doit aider a prendre

Une mauvaise question produit une recherche brillante mais inutile.

### 2. Definir le standard de preuve

Classer le sujet:

- faible risque: synthese rapide acceptable
- risque moyen: recherche sourcee standard
- risque eleve: sources primaires, verification croisee, marquage strict

Par defaut, tout ce qui touche legal, conformite, fiscalite ou workflow client reglemente est en risque eleve.

### 3. Recolter les sources

Chercher d'abord:

- organismes officiels
- textes de loi et reglements
- documentation fournisseur
- guides professionnels reconnus

Ajouter ensuite:

- analyses sectorielles
- presse specialisee
- retours terrain clairement identifies

### 4. Trier et annoter

Pour chaque source utile, noter:

- ce qu'elle etablit vraiment
- sa date
- son autorite
- ses limites

Ne pas melanger trop tot la collecte et la narration.

### 5. Produire le premier squelette

Construire la recherche avec:

1. question
2. resume executif
3. faits verifies
4. analyse
5. risques
6. recommandations
7. sources

### 6. Marquer le niveau de certitude

Chaque point important doit etre classe:

- `VERIFIE`
- `INFERENCE`
- `A_CONFIRMER`

### 7. Passer la checklist QA

Utiliser [RESEARCH_QA_CHECKLIST.md](/Users/Bookkeeping/SAAS%20-%20SAFE%2002/docs/research/RESEARCH_QA_CHECKLIST.md:1) integralement.

### 8. Produire deux versions si utile

Quand le sujet est gros, livrer:

- une version executive courte
- une version longue de reference

Si une phrase est impressionnante mais pas clairement supportee, on la reecrit ou on la retire.
